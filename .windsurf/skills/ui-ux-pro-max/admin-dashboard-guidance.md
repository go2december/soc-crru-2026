# Admin Dashboard Guidance

## Use This Mode For

- CRUD index pages
- Create/edit forms
- Dashboard cards and summaries
- Confirmation dialogs
- Data-table style admin screens

## Implementation Rules

- Use shadcn/ui components from `frontend/components/ui/`.
- Preserve the current admin layout, auth flow, and route structure.
- Keep admin UI professional, compact, and task-oriented.
- Prioritize clarity of actions, error states, confirmation states, and density of information.

## Validation Focus

- No DaisyUI components in admin pages
- Existing auth/callback flows still work
- CRUD pages preserve list/create/edit/delete usability
