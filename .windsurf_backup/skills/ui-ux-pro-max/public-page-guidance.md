# Public Page Guidance

## Use This Mode For

- Landing sections
- Public content pages
- Detail pages
- Marketing-style layouts
- Responsive public-facing redesigns

## Implementation Rules

- Use DaisyUI classes and the project's public component patterns.
- Reuse components from `frontend/components/` before creating new ones.
- Prefer server-friendly Next.js App Router patterns.
- Keep public pages readable, accessible, and consistent with existing navigation and spacing patterns.

## Validation Focus

- Visual consistency with Scholar or Chiang Rai branding
- Responsive behavior on mobile/tablet/desktop
- No accidental imports from `frontend/components/ui/`
