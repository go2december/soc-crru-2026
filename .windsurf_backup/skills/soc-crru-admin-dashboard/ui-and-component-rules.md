# UI and Component Rules

## shadcn/ui Rules

- Use shadcn/ui components from `frontend/components/ui/` in admin layouts and pages.
- Keep admin pages visually consistent with the professional slate theme.
- Reuse existing admin patterns before creating new components.

## Anti-Patterns

- Do not use DaisyUI components in admin pages.
- Do not mix DaisyUI classes with shadcn/ui primitives in the same admin component.
- Do not move admin-only UI into public component folders.
